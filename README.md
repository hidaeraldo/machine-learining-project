# Predicting 30-Day Hospital Readmission for Diabetic Patients

Binary classification project using the **Diabetes 130-US Hospitals (1999-2008)** dataset from the UCI Machine Learning Repository. The goal is to predict whether a diabetic patient will be readmitted within 30 days of hospital discharge.

## Dataset

- **Source:** [UCI ML Repository – Diabetes 130-US Hospitals](https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008)
- **Citation:** Strack, B., DeShazo, J.P., et al. "Impact of HbA1c Measurement on Hospital Readmission Rates." *BioMed Research International*, 2014.
- **Size:** 101,766 encounters × 50 features
- **Target:** `readmitted` → binarized to 1 (readmitted <30 days) vs 0 (otherwise)

## Setup

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd Project
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Place the dataset files in `data/raw/`:
   - `diabetic_data.csv` (download from the UCI link above)
   - `IDs_mapping.csv` (included in the repository)

4. Run the notebook:
   ```bash
   cd notebooks
   jupyter notebook phase1_eda_preprocessing.ipynb
   ```

## Phase 1: EDA, Preprocessing & Feature Engineering

The notebook covers the complete Phase 1 pipeline:

1. **Problem Definition** – Clinical motivation, dataset description, feature groups
2. **Load & Inspect** – Shape, dtypes, summary statistics
3. **Missing Values** – Analysis, classification (MCAR/MAR/MNAR), drop/impute decisions
4. **Duplicates** – Patient deduplication to prevent data leakage
5. **Target Analysis** – Binarization, class imbalance documentation
6. **Univariate Analysis** – Histograms, skewness, categorical distributions, near-zero variance
7. **Bivariate Analysis** – Boxplots, stacked bars, scatter plots
8. **Outlier Detection** – IQR method, clinical interpretation
9. **Correlation Analysis** – Heatmap, target correlations, multicollinearity check
10. **EDA Summary** – 7 key findings
11. **Preprocessing** – Cleaning, encoding, imputation, one-hot encoding
12. **Feature Engineering** – New features, binning, log transforms, variance filter
13. **Train-Test Split** – 80/20 stratified, scaled + unscaled outputs
14. **Completion Summary** – Decision recap, lab requirement checklist
